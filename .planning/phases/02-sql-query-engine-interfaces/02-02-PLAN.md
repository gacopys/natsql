### Contract this plan implements

Plan 01 created the `natsql/query` package. This plan creates `natsql/engine.Engine.Query()` which wraps it, and `natsql/transport` package handlers that call `Engine.Query()`.

The transport package signature:
```go
package transport

import "natsql/engine"

// QueryHandler is the interface Engine satisfies for transport usage.
// The transport package receives this interface, not the concrete Engine type.
type QueryHandler interface {
    Query(ctx context.Context, sql string) *query.QueryResult
}

// RegisterNATSHandler subscribes to natsql.query and responds with query results.
// sub is *nats.Conn (for Subscribe), engine is the query handler.
func RegisterNATSHandler(nc *nats.Conn, handler QueryHandler) (*nats.Subscription, error)

// RegisterHTTPHandler mounts POST /api/v1/query on the chi router.
func RegisterHTTPHandler(router chi.Router, handler QueryHandler)
```

    **Step 2: Implement `Engine.Query()`**

    ```go
    import "natsql/query"

    func (e *Engine) Query(ctx context.Context, sql string) *query.QueryResult {
        // 1. Parse SQL
        q, err := query.Parse(sql)
        if err != nil {
            errStr := err.Error()
            return &query.QueryResult{Results: nil, Error: &errStr}
        }

        // 2. Load schema from KV (per D-27 — no cache, always fresh)
        schema, err := kv.LoadSchema(ctx, e.kv, q.From)
        if err != nil {
            errStr := fmt.Sprintf("error loading schema: %v", err)
            return &query.QueryResult{Results: nil, Error: &errStr}
        }
        if schema == nil {
            errStr := fmt.Sprintf("view %q not found", q.From)  // D-42
            return &query.QueryResult{Results: nil, Error: &errStr}
        }

        // 3. Validate against schema
        if err := query.Validate(q, schema); err != nil {
            errStr := err.Error()
            return &query.QueryResult{Results: nil, Error: &errStr}
        }

        // 4. Build plan
        plan, err := query.BuildPlan(q, schema)
        if err != nil {
            errStr := err.Error()
            return &query.QueryResult{Results: nil, Error: &errStr}
        }

        // 5. Execute plan
        rows, err := plan.Execute(ctx, e.kv)
        if err != nil {
            errStr := err.Error()
            return &query.QueryResult{Results: nil, Error: &errStr}
        }

        // 6. Return results
        return &query.QueryResult{Results: rows, Error: nil}
    }
    ```

    **Step 2: Create `natsql/transport/nats.go`**

    ```go
    package transport

    import (
        "context"
        "encoding/json"
        "github.com/nats-io/nats.go"
        "natsql/query"
    )

    // QueryHandler is the interface transports use to execute queries.
    type QueryHandler interface {
        Query(ctx context.Context, sql string) *query.QueryResult
    }

    // RegisterNATSHandler subscribes to natsql.query (D-34) and responds
    // with the JSON query result. The request body is the raw SQL string (D-35).
    // Returns the subscription for lifecycle management.
    func RegisterNATSHandler(nc *nats.Conn, handler QueryHandler) (*nats.Subscription, error) {
        sub, err := nc.Subscribe("natsql.query", func(msg *nats.Msg) {
            sql := string(msg.Data)
            result := handler.Query(context.Background(), sql)
            data, _ := json.Marshal(result)
            msg.Respond(data)
        })
        if err != nil {
            return nil, fmt.Errorf("subscribing to natsql.query: %w", err)
        }
        nc.Flush()
        return sub, nil
    }
    ```

    **Step 3: Create `natsql/transport/http.go`**

    ```go
    package transport

    import (
        "context"
        "encoding/json"
        "net/http"
        "github.com/go-chi/chi/v5"
        "github.com/go-chi/chi/v5/middleware"
        "natsql/query"
    )

    type QueryRequest struct {
        SQL string `json:"sql"`
    }

    // RegisterHTTPHandler mounts the query endpoint on the chi router (D-37).
    func RegisterHTTPHandler(router chi.Router, handler QueryHandler) {
        router.Post("/api/v1/query", func(w http.ResponseWriter, r *http.Request) {
            var req QueryRequest
            if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
                http.Error(w, `{"results":[],"error":"invalid JSON body"}`, http.StatusBadRequest)
                return
            }
            result := handler.Query(r.Context(), req.SQL)
            w.Header().Set("Content-Type", "application/json")
            json.NewEncoder(w).Encode(result)
        })
    }

    // NewRouter creates a chi router with standard middleware for the query API.
    func NewRouter() chi.Router {
        r := chi.NewRouter()
        r.Use(middleware.Logger)
        r.Use(middleware.Recoverer)
        r.Use(middleware.Timeout(30 * time.Second))
        return r
    }
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| NATS message→transport handler | Untrusted SQL from any NATS client on the same JetStream domain |
| HTTP request→transport handler | Untrusted HTTP request from any network-accessible client |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Tampering | NATS handler | accept | Any NATS client can send queries. No auth in v1. NATS-based auth (token/Nkey) is caller's responsibility. Document in transport layer that auth is not enforced. |
| T-02-06 | Tampering | HTTP handler | accept | HTTP server binds to all interfaces by default. No auth in v1. Recommend reverse proxy for auth. Serve on localhost only for dev (127.0.0.1:8080 default). |
| T-02-07 | DoS | HTTP body | mitigate | chi middleware.Timeout(30s) prevents long-running queries from blocking. No explicit body size limit in v1 — rely on NATS message size defaults (1MB). |
| T-02-08 | DoS | NATS subscription | accept | NATS subscription runs in the engine process. Large JSON responses could consume memory. Query results bounded by KV row size (default 64KB) + LIMIT. |
| T-02-09 | Information Disclosure | HTTP error responses | accept | Error messages include SQL text for debugging (D-42 note). No sensitive data leakage as DB credentials are not in SQL queries. |

</threat_model>
