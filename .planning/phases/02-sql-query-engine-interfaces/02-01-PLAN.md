package query

// Query represents a parsed SELECT statement.
type ValidatedQuery struct {
    Select     []string    // column names; nil = "*"
    From       string      // view name
    Where      []Condition // AND-connected conditions
    Limit      int         // 0 = no limit
}

type Condition struct {
    Column string
    Op     Op
    Value  any
}

type Op string
const OpEq  Op = "="
const OpNeq Op = "!="
const OpIn  Op = "in"

// Plan is the execution plan interface.
type Plan interface {
    Execute(ctx context.Context, kv jetstream.KeyValue) ([]map[string]any, error)
}

// PKLookupPlan is used when WHERE has pk_col = value.
type PKLookupPlan struct { ViewName string; PkValue string; Columns []string }

// FullScanPlan is used for all other queries (non-PK WHERE, no WHERE).
type FullScanPlan struct { ViewName string; Columns []string; Where []Condition; Limit int }

// QueryResult is the JSON-serializable output envelope per D-29.
type QueryResult struct {
    Results []map[string]any `json:"results"`
    Error   *string         `json:"error"`
}

func Parse(sql string) (*ValidatedQuery, error)
func Validate(q *ValidatedQuery, schema *kv.ViewSchema) error
func BuildPlan(q *ValidatedQuery, schema *kv.ViewSchema) (Plan, error)

    Use `vitess.io/vitess/go/vt/sqlparser`:
    ```go
    import "vitess.io/vitess/go/vt/sqlparser"

    func Parse(sql string) (*ValidatedQuery, error) {
        stmt, err := sqlparser.Parse(sql)
        if err != nil {
            return nil, fmt.Errorf("parse error: %w", err)
        }
        sel, ok := stmt.(*sqlparser.Select)
        if !ok {
            return nil, errors.New("only SELECT statements are supported")
        }
        // Extract From (assume single table)
        // Extract SelectExprs (column names or *StarExpr)
        // Extract Where (ComparisonExpr with Left, Operator, Right)
        // Extract Limit
        return q, nil
    }
    ```

    Key vitess AST traversal rules:
    - `sel.From` → `TableExprs` → first is `*sqlparser.AliasedTableExpr` → `Expr` is `sqlparser.TableName` → `Name.String()` = view name
    - `sel.SelectExprs` → each is `*sqlparser.AliasedExpr` → `Expr` is `sqlparser.ColIdent` → `String()` = column name. If it's `*sqlparser.StarExpr`, columns = nil (meaning "*")
    - `sel.Where` → `*sqlparser.Where` → `Expr` is `*sqlparser.ComparisonExpr` (for v1, only =, !=, IN). `Left` = `*sqlparser.ColIdent`, `Right` = `*sqlparser.SQLVal` (for strings) or `*sqlparser.ListArg` (for IN)
    - `sel.Limit` → `*sqlparser.Limit` → `Rowcount` = `*sqlparser.SQLVal`
    - FROM subqueries or JOINs: return error "only simple SELECT FROM view is supported"

    Handling SQLVal types:
    - `sqlparser.StrVal` → string
    - `sqlparser.IntVal` → int64
    - `sqlparser.FloatVal` → float64

    For IN operator: `Right` is `*sqlparser.ValTuple` containing multiple SQLVal.

    ```go
    func BuildPlan(q *ValidatedQuery, schema *kv.ViewSchema) (Plan, error) {
        // If any WHERE condition matches a PK column with OpEq → PKLookupPlan
        // For multiple WHERE: if any is PK=, use PKLookupPlan. Remaining conditions should be applied
        //   as a post-filter in the executor after fetching the row. (PK always returns 0-1 rows.)
        // Otherwise → FullScanPlan
    }
    ```

    PK columns are in `schema.KeyFields` (list of key field names from the view config).
    A condition is PK-equality if: `cond.Op == OpEq` AND `cond.Column` is in `schema.KeyFields`.

    For PKLookupPlan with multiple PK fields (composite key), combine values. The PkValue should look like what `natsql/kv` produces — but the query executor needs to construct the PK value from one or more condition values.

    **Composite key handling**: If `len(schema.KeyFields) > 1`:
    - A PK lookup requires ALL key fields to have equality conditions
    - If all are present, join them with the schema's `KeySeparator` (default `|`)
    - If not all present, fall back to FullScanPlan
    - This is documented in the plan but unlikely to be tested in v1

    **Row projection (`projectRow` helper):**
    ```go
    func projectRow(data []byte, columns []string) (map[string]any, error) {
        var row map[string]any
        json.Unmarshal(data, &row)
        if columns == nil { return row, nil }  // SELECT *
        // Filter to only requested columns
        projected := make(map[string]any, len(columns))
        for _, col := range columns {
            projected[col] = row[col]  // nil if missing (D-31)
        }
        return projected, nil
    }
    ```

    **WHERE filter (`filterRow` helper):**
    ```go
    func filterRow(row map[string]any, conditions []Condition) bool {
        for _, c := range conditions {
            val := row[c.Column]
            switch c.Op {
            case OpEq:
                if fmt.Sprint(val) != fmt.Sprint(c.Value) { return false }
            case OpNeq:
                if fmt.Sprint(val) == fmt.Sprint(c.Value) { return false }
            case OpIn:
                values, ok := c.Value.([]any)
                if !ok { return false }
                found := false
                for _, v := range values {
                    if fmt.Sprint(val) == fmt.Sprint(v) { found = true; break }
                }
                if !found { return false }
            }
        }
        return true
    }
    ```

    Use `fmt.Sprint` for comparison rather than type assertion — values from JSON unmarshaling and parsed conditions may have different concrete types (e.g., float64 vs int64).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→SQL parser | Untrusted SQL string from NATS message or HTTP body enters SQL parser |
| SQL parser→KV executor | Parsed and validated query struct flows to KV executor |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | vitess SQL parser | mitigate | Input is SQL string. vitess is battle-tested against SQL injection. Our dialect is SELECT-only with no DML, limiting attack surface. Malformed SQL returns parse error, never executed. |
| T-02-02 | DoS | FullScanPlan executor | mitigate | `ListKeys()` returns ALL keys in bucket. Document O(n) performance. For v1, accept this risk. Future: per-view buckets or index enforcement. |
| T-02-03 | DoS | vitess parser | mitigate | vitess has input-size protection. Extremely long SQL strings limited by NATS message size (1MB default) or HTTP body limits. |
| T-02-04 | Information Disclosure | QueryResult | accept | Query results contain materialized data from KV. Authorization is out of scope for v1 — NATS and HTTP transport level security is caller's responsibility. |
</threat_model>
