**User decisions implemented in this plan:**
- D-01: Full config from day one (indexes, retry, DLQ, bucket, event mapping)
- D-02: Single YAML file supports multiple views (`views:` array)
- D-03: Column mapping uses JSON path notation (`user.id` → nested)
- D-04: Composite keys via `key_fields: [field1, field2]` with configurable separator (default `|`)
- D-05: Indexes field in config schema (forward compatibility, ignored by materializer)
- D-06: Single KV bucket (`natsql-views`) for all views
- D-07: Row key format `{view_name}:{pk_value}`
- D-08: View schema stored in KV at `schemas:{view_name}`
- D-09: Explicit consumer config (batch size, MaxDeliver, ack wait)
- D-15: Column types: `string`, `number`, `boolean`, `timestamp`
</context>

    1. Create `natsql/kv/kv.go` with package `kv`:

    ```go
    package kv

    import (
        "context"
        "encoding/json"
        "fmt"
        "strings"

        "github.com/nats-io/nats.go/jetstream"
    )

    const (
        DefaultBucket = "natsql-views"  // D-06: single bucket for all views
        SchemaPrefix  = "schemas:"       // D-08: schema key prefix
    )
    ```

    - `func InitBucket(ctx context.Context, js jetstream.JetStream, replicas int) (jetstream.KeyValue, error)`
      - Calls `js.CreateOrUpdateKeyValue(ctx, jetstream.KeyValueConfig{Bucket: DefaultBucket, Storage: jetstream.FileStorage, Replicas: replicas})`
      - Returns the KeyValue handle

    - `func PkKey(viewName, pkValue string) string`
      - Returns `"{viewName}:{pkValue}"` per D-07
      - Example: `PkKey("users", "abc123")` → `"users:abc123"`

    - `func SchemaKey(viewName string) string`
      - Returns `"schemas:{viewName}"` per D-08
      - Example: `SchemaKey("users")` → `"schemas:users"`

    - `func StoreSchema(ctx context.Context, kv jetstream.KeyValue, viewName string, schema *natsql_import.ViewSchema) error`
      - Marshals schema to JSON
      - Puts to KV at SchemaKey(viewName)
      - Uses `kv.Put()` — no CAS needed for initial schema write (new views always create)

    - `func LoadSchema(ctx context.Context, kv jetstream.KeyValue, viewName string) (*natsql_import.ViewSchema, error)`
      - Gets from KV at SchemaKey(viewName)
      - If key not found, returns (nil, nil) — not an error
      - Unmarshals JSON into ViewSchema

    - `func EncodePKValue(val any) string`
      - Returns string representation for use in KV key
      - string → as-is (but must not contain `:` — panic if it does)
      - int64/float64 → fmt.Sprint(val) (no zero-padding needed for PK keys since they're direct lookups)
      - bool → "true" / "false"
      - nil → "" (empty — null PK is invalid, should be caught by validation)
      - **Note:** This is for the PK value encoding in the key. Range-sortable encoding (zero-padded hex) is for index values (Phase 2) per ARCHITECTURE.md §3.2.

    - `func MustInitBucket(...)` — convenience wrapper around InitBucket that panics on error (for use in main/startup).
