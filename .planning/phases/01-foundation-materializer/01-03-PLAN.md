**Processing loop pattern (from ARCHITECTURE.md §2.5):**
```
for {
    select {
    case msg := <-consumer.Messages():
        mut, err := mapper.MapRow(msg)
        if err == nil && mut != nil {
            err = writer.Apply(ctx, mut)
        }
        if err == nil {
            msg.Ack()
        } else if errors.Is(err, ErrMalformedEvent) {
            // Send to DLQ, then ack
            msg.Ack()
        } else {
            msg.Nak()
        }
    case <-ctx.Done():
        return ctx.Err()
    }
}
```

    - `func Run(ctx context.Context, js jetstream.JetStream, viewCfg *ViewConfig, kv jetstream.KeyValue, dlqStream jetstream.Stream, logger *slog.Logger) error`
      - **Purpose:** Run the materializer for a single view. Blocks until ctx is cancelled.
      - **Steps:**
        1. Create consumer: `cons, err := SetupConsumer(ctx, js, viewCfg.SourceStream, viewCfg.Name, viewCfg.Consumer)`
        2. Create mapper: `mapper, err := NewMapper(viewCfg)`
        3. Create writer: `writer := NewWriter(kv, viewCfg.Name)`
        4. Store schema in KV: `schema := viewCfg.BuildSchema(); kv.StoreSchema(ctx, kv, viewCfg.Name, schema)`
        5. Start heartbeat ticker (every 60 seconds, logs "processed N events"):
           ```go
           heartbeat := time.NewTicker(60 * time.Second)
           defer heartbeat.Stop()
           ```
        6. **Processing loop** (core of the materializer):
           ```go
           var eventCount int64
           msgChan := cons.Messages()

           for {
               select {
               case msg := <-msgChan.Messages():
                   eventCount++
                   // Map event → RowMutation
                   mut, mapErr := mapper.MapRow(msg)
                   if mapErr != nil {
                       if errors.Is(mapErr, ErrMalformedEvent) {
                           // D-12: Malformed → DLQ + ack
                           publishToDLQ(ctx, dlqStream, msg, viewCfg.Name, mapErr)
                           msg.Ack() // D-12: ack to skip, don't block
                           logger.Warn("skipped malformed event", "seq", msg.Metadata().Sequence.Stream, "error", mapErr)
                           continue
                       }
                       // Unknown error → Nak for retry
                       msg.Nak()
                       logger.Error("unexpected mapper error", "error", mapErr)
                       continue
                   }

                   // Apply to KV
                   if mut != nil {
                       if writeErr := writer.Apply(ctx, mut); writeErr != nil {
                           // D-14: Persistent write error → DLQ, don't stall
                           publishToDLQ(ctx, dlqStream, msg, viewCfg.Name, writeErr)
                           msg.Ack() // ack to move past, don't block on bad state
                           logger.Error("write failed, sent to DLQ", "seq", msg.Metadata().Sequence.Stream, "error", writeErr)
                           continue
                       }
                   }

                   // D-11: Ack only after successful KV write
                   msg.Ack()

               case <-heartbeat.C:
                   logger.Info("materializer heartbeat", "view", viewCfg.Name, "events_processed", eventCount)

               case <-ctx.Done():
                   logger.Info("materializer shutting down", "view", viewCfg.Name, "events_processed", eventCount)
                   return ctx.Err()
               }
           }
           ```

      - **Error handling rules** (per ARCHITECTURE.md §2.6 and D-14):
        - Malformed event (ErrMalformedEvent): publish to DLQ, ack, continue
        - KV write failure: publish original to DLQ, ack, continue (don't stall)
        - Context cancelled: return immediately
        - Any other error: publish to DLQ and ack (D-14: never stall the consumer)

    ```go
    package main

    import (
        "context"
        "log/slog"
        "os"
        "os/signal"
        "syscall"

        "github.com/nats-io/nats.go"
        "github.com/nats-io/nats.go/jetstream"
        "natsql"
    )

    func main() {
        // 1. Parse config path from args (default: "config.yaml")
        cfgPath := "config.yaml"
        if len(os.Args) > 1 {
            cfgPath = os.Args[1]
        }

        // 2. Load config
        cfg, err := natsql.LoadConfig(cfgPath)
        if err != nil {
            slog.Error("failed to load config", "error", err)
            os.Exit(1)
        }

        // 3. Connect to NATS
        nc, err := nats.Connect(nats.DefaultURL)
        if err != nil {
            slog.Error("failed to connect to NATS", "error", err)
            os.Exit(1)
        }
        defer nc.Close()

        js, err := jetstream.New(nc)
        if err != nil {
            slog.Error("failed to create JetStream context", "error", err)
            os.Exit(1)
        }

        // 4. Create engine
        engine, err := natsql.New(js, cfg)
        if err != nil {
            slog.Error("failed to create engine", "error", err)
            os.Exit(1)
        }

        // 5. Start engine
        ctx, cancel := context.WithCancel(context.Background())
        defer cancel()

        if err := engine.Start(ctx); err != nil {
            slog.Error("failed to start engine", "error", err)
            os.Exit(1)
        }

        // 6. Wait for SIGINT/SIGTERM
        sigCh := make(chan os.Signal, 1)
        signal.Notify(sigCh, syscall.SIGINT, syscall.SIGTERM)
        sig := <-sigCh
        slog.Info("received signal, shutting down", "signal", sig)

        // 7. Graceful shutdown
        if err := engine.Close(); err != nil {
            slog.Error("error during shutdown", "error", err)
            os.Exit(1)
        }
        slog.Info("engine shut down gracefully")
    }
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| YAML config file → Engine.New | Untrusted file input (same as Plan 01) |
| JetStream event → materializer.Run | Untrusted external event data |
| Materializer → DLQ stream | Internal system boundary (trusted) |
| Engine → OS signals (SIGINT/SIGTERM) | OS-level lifecycle (trusted) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Denial of Service | Infinite malformed events flooding DLQ | mitigate | DLQ stream has MaxAge=7d (auto-evicts old messages). Rate-limit DLQ publish logging (log at most once per 10 seconds per view). |
| T-03-02 | Denial of Service | Event processing slower than event production | accept | v1 processes sequentially. Document that materializer keeps up with NATS KV throughput (~1K writes/sec). Use larger batch sizes to compensate. |
| T-03-03 | Tampering | DLQ envelope data | accept | DLQ is internal system stream. Access controlled by NATS auth. |
| T-03-04 | Repudiation | Missing audit trail for skipped events | accept | DLQ preserves skipped events. Stream is the authoritative changelog. |
| T-03-05 | Denial of Service | Engine Start called without proper NATS connection | mitigate | New() validates config only. Start() returns error if stream/consumer setup fails. Early exit before goroutine launch on failure. |
</threat_model>
