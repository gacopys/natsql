# Requirements: natsql v1.2

**Archived:** 2026-07-01
**Milestone:** v1.2 Code Review Remediation
**Status:** ✅ All 26 requirements satisfied

## Requirements

### Verification

- [x] **VER-01**: Each finding in cr.md is examined against source code and confirmed or dismissed with documented reasoning ✓

### Foundation — PK Encoding, SQL Rejection, Config Validation

- [x] **FND-01**: PK values are encoded by a single canonical function used by both write (materializer) and read (query planner/executor) paths, eliminating double-sanitization and lookup key mismatch ✓
- [x] **FND-02**: Unsupported SQL constructs (ORDER BY, DISTINCT, GROUP BY, HAVING, aggregations, subqueries) are explicitly rejected with an error instead of silently mishandled ✓
- [x] **FND-03**: Config validation cross-references `key_fields` against declared `primary_key` columns, verifies uniqueness of view names and column names, and rejects invalid configurations at load time ✓

### Materializer — Ordered Processing, Error Classification, Consumer Config

- [x] **MAT-01**: Materializer processes events from a single durable consumer in stream order, preserving JetStream's per-subject ordering guarantee at the KV write boundary ✓
- [x] **MAT-02**: Materializer classifies KV write errors: transient (connection/leader election) → NAK with backoff for redelivery, terminal (bad data/config) → DLQ + Ack, malformed input → DLQ + Ack ✓
- [x] **MAT-03**: Durable consumers do not have an InactiveThreshold that could cause automatic deletion after downtime ✓
- [x] **MAT-04**: Consumer BatchSize setting is renamed to MaxAckPending to match its actual behavior ✓

### Query Engine — Predicates, Projection, Precision, Scans

- [x] **QENG-01**: All WHERE conditions are retained as post-filters, including conditions on PK columns, preventing contradictory or duplicate PK predicates from producing wrong results ✓
- [x] **QENG-02**: `SELECT *` returns only schema-declared columns, excluding internal `_meta` fields ✓
- [x] **QENG-03**: Query executor uses `json.Decoder.UseNumber()` for consistent numeric comparison, matching the materializer's precision handling for large integers above 2^53 ✓
- [x] **QENG-04**: Full-scan queries for a single view do not pay the cost of scanning all other views' KV keys — WatchAll + HasPrefix prefix filter limits scan ✓

### Engine Lifecycle — Port Config, Startup Error Propagation

- [x] **LIFE-01**: HTTP server port is initialized from `cfg.HTTP.Port` in engine constructors, not hardcoded to 8080 ✓
- [x] **LIFE-02**: Engine.Start propagates startup errors synchronously: HTTP listen failures, materializer init failures, and NATS handler registration failures prevent the engine from reporting as started ✓

### Transport — Stream Creation, HTTP Robustness, NATS Error Handling

- [x] **TRN-01**: CLI stream creation respects configured `source_subject` and does not mutate existing external streams without explicit opt-in ✓
- [x] **TRN-02**: HTTP JSON handler uses `*http.MaxBytesError` via `errors.As` for body-size detection and rejects trailing non-whitespace data ✓
- [x] **TRN-03**: NATS transport checks and surfaces `nc.Flush()` and `msg.Respond()` errors instead of ignoring them ✓
- [x] **TRN-04**: Error message in executor.go is corrected from "marshaling row" to "unmarshaling row" ✓

### Cleanup — Dead Code, Formatting, Tests, Docs, Examples

- [x] **CLN-01**: `$.` prefix notation in field paths is fully supported via `strings.TrimPrefix` in extractNestedField ✓
- [x] **CLN-02**: Index configuration produces a clear validation error at load time ✓
- [x] **CLN-03**: Missing delete/tombstone semantics documented as a known gap with planned future support (v2) ✓
- [x] **CLN-04**: All examples check errors properly and avoid lifecycle ownership hazards ✓
- [x] **CLN-05**: Unused symbols removed (SchemaPrefix, ErrSkipAndAck, Stats.LastError, dlqStream parameter, EncodePKValue, MustInitBucket); one canonical PK encoding path remains ✓
- [x] **CLN-06**: All Go source files formatted with `gofmt -w`; CI enforces formatting ✓
- [x] **CLN-07**: Embedded NATS setup and stream creation helpers deduplicated into `internal/testutil` ✓
- [x] **CLN-08**: README and SQL dialect documentation updated to accurately reflect implemented features including LIMIT support ✓
