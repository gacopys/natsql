# Milestone v1.2: Code Review Remediation

**Status:** ✅ SHIPPED 2026-07-01
**Phases:** 8-11
**Total Plans:** 12 plans (4+3+2+3)

## Overview

Remediated all 25 findings from the comprehensive code review (cr.md), achieving 100% project correctness. Fixed critical correctness bugs (concurrent state corruption, PK sanitization inconsistency, contradictory PK predicates), high-severity issues (meta field leakage, unsupported SQL rejection, config validation, startup error propagation, precision loss, transient failure handling), and medium/low issues (prefix support, index config, delete semantics documentation, example hygiene, dead code removal, test helper deduplication).

## Key Accomplishments

1. **Verification baseline (Phase 8):** All 25 code review findings verified against source code — 25 confirmed, 0 dismissed. Config cross-validation, SQL parser rejection of unsupported constructs, and canonical PK encoder (`BuildPkKey`) implemented.

2. **Sequential processing (Phase 9):** Removed 16-goroutine worker pool — preserves JetStream per-subject ordering at the KV write boundary. Added error classification (transient→NAK, terminal→DLQ), InactiveThreshold removal, BatchSize→MaxAckPending rename.

3. **Synchronous startup (Phase 9):** Engine.Start propagates all startup errors synchronously — HTTP listen failures, materializer init failures, and NATS handler registration failures now prevent the engine from reporting as started.

4. **Query engine correctness (Phase 10):** All WHERE conditions retained as post-filters with contradictory PK short-circuit. `SELECT *` excludes internal `_meta` fields. Large integer precision preserved via `json.Decoder.UseNumber()` with `json.Number` comparison support.

5. **Transport robustness (Phase 10):** HTTP uses `errors.As` for MaxBytesError detection with trailing data rejection. NATS surfaces Flush/Respond errors. CLI supports `--create-streams` flag with `source_subject` respect.

6. **Cleanup & CI overhaul (Phase 11):** Dead code removed, `$.` prefix supported, index config validated, test helpers deduplicated into `internal/testutil`, examples hardened, SQL_DIALECT.md created. Plus: golangci-lint config with 40 linters, govulncheck, 5 parallel CI workflows with README badges, all vulnerabilities fixed (Go 1.26.4).

## Milestone Summary

### Key Decisions

- D-07/D-08: UseNumber in both PK and FullScan decode paths (preserves large integer precision)
- D-09/D-10: convertJSONNumber normalizes json.Number for valuesEqual comparison; WHERE literals remain as native types
- D-11 (Phase 10): WatchAll + HasPrefix prefix filter fallback (Watch(prefix) can't handle `/` separators)
- D-16: errors.As(err, &maxBytesErr) for MaxBytesError detection (replaced string comparison)
- D-17/D-18: json.Decoder double-decode pattern for trailing data detection

### Tech Debt Deferred

- Range scans (>, <, >=, <=) — deferred to v2
- Secondary indexes — rejected at config validation, not implemented
- Delete/tombstone semantics — documented as known gap
- WatchAll + HasPrefix fallback — structurally constrained by KV key separator choice
